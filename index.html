Floodead — 半淹办公楼生存恐怖

把 FlooDead 换皮成「Wanaka 办公楼被暴雨半淹、同事被污染」的三章式第一人称恐怖。

checkpoint chk_0Mm2ESM6aNXVR0m 编辑器 / 可玩 →

做了什么

全功能实现。玩家是无名员工，探索半淹大楼、面对变成感染体的真实同事——可以打倒他们，也可以用血清救回；救的人越多结局越好。第三章 Boss 是柴峻洋本人，全员用真实 wall 成员名。

核心循环还原度

  • ✅三章第一人称探索半淹办公楼（办公层 4800² → 地下机房 → 研究层污染源）
  • ✅手持武器反击：F 攻击掉血击杀，掉工牌揭示真实队友身份
  • ✅多怪群同时 AI（findAll + WeakMap per-enemy 状态）
  • ✅道德抉择：击杀 vs 血清救助，救助数驱动 4 档结局（归途≥5 / 救赎≥3 / 微光≥1 / 幸存 0）
  • ✅蜿蜒地图：二三章交错隔断墙逼出 S 形，怪一段一个逐个遭遇
  • ✅恐怖节奏：突然逼近 jump scare + 近距离心跳越近越急 + 偶发跳电
  • ✅音效：击打 / 对话 / 恐怖追逐 drone / 心跳 / 稀疏滴水（numpy 自制 + S3 上传）
  • ✅离职同事救场彩蛋（跨三章）：濒危时戴炜恒 / 赵美晨瞬移登场压制怪 + 回血
  • ◐美术：已换 CC-BY server rack + 自制音频；其余多为 catalog 占位（动画角色不换以保 clip）
  • ✗真窗口平衡验证：难度 / 手感 / 像不像原作需导演真人试玩定夺

踩的坑

坑现象 + 根因 + 修复发现
assetStoreId 音频不播assetStoreId 绑的 obj.Audio 无声 → 全改 bundled asset: 资源真窗口
碰撞顶穿地动态玩家胶囊把 kinematic 怪 y 顶穿地 → 每帧 m.y=90 锁定真窗口
上半屏黑开顶房间露暗背景 + far plane → 加天花板 + far=10000真窗口
自定义资产 deploy 不传内容寻址，本地 assets/ 不随 deploy 传 → aws s3api put-object紫块
短 mp3 循环变很快心跳短音循环 = 频率过高 → 改定时单声、按距离拉间隔真窗口
手动刷 CLI token 断续期refresh 接口会轮换 token；没存回就断链 → 只跑 wanaka login、别碰 credentials（30 天自动续期）每小时掉登录

与原作差距

  • ◐美术保真度：仍以 catalog 模型为主；水面 / 灯光 / 家具 / 尸体道具可再提升（Sketchfab→S3 管线已通、就差批量做）
  • ◐动画角色不换模型：换了会丢 Zombie_* 动画 clip、破坏框架，故保留 catalog 僵尸
  • ✗平衡 / 像不像原作：导演真人判断，需完整试玩定夺
  • ✗Ch1 未做蜿蜒；Boss 战为掉血近战；敌人单一种类；无章节间存档
